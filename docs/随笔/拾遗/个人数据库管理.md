个人数据在随着时间不断地积累，很多文件分散在各个文件夹里，每次查找的时候都很费劲，这就是新建文件的时候不做好规划的后果。另外，随着设备越来越多，就越发觉得数据安全的重要性，一旦保存数据的设备坏了，数据就全丢了，不敢想象这种情况的发生。因此下面着手整理了一下现有的所有数据。

## 数据归档
给数据归档类，就是给数据找一个合理的分类方式。

我之前给数据归档是粗略地按数据的类型执行的，比如图片、文档、代码。但是问题显而易见，很多数据需要放在一起才有意义，比如某次项目的所有资料。

另一种思路就是安装领域划分，比如工作、学习、娱乐，每个类别下面再细分子类，这是很符合直觉的一种方式，就像图书馆的书籍管理一样。

### 一级分类
一级分类是主干分类，是具体的领域，一个文件属于哪个领域，是需要思考的。当前有下面几个类别：

```plain
个人数字资产中心
├── 00-待处理与中转
├── 01-工作与产出
├── 02-学习与知识
├── 03-财产与资源
├── 04-关系与记录
├── 05-娱乐与收藏
└── 06-系统与工具
```

### 二级分类
二级分类在一级的基础上进一步细化，对于一个文件来说，是我们下意识想到的类别。

下面的类别肯定是不全面的，可以在归档的过程中逐步完善。

## 数据命名
数据命名同样是必不可少的一环。

`1.txt`、`2.txt`这种临时命名的文件，还有从微信或者 QQ 接受的图片文件，文件名往往都是一串随机字符串，在不打开的情况下，完全不知道里面是什么，是否必要保留，如果想要检索那更是难上加难。

文件命名公式：

```plain
YYYYMMDD_项目/主题_关键词_版本.扩展名
```

## 数据备份
最重要的就是数据备份，这是数据安全的前提。

数据备份有很多种方式，PC 自带的硬盘、移动硬盘、网盘、NAS 等，现阶段我是没有 NAS 的，前面几个倒是都有，可以好好利用起来。

下面是数据备份的思路：

```plain
PC硬盘：主工作区
移动硬盘：本地备份库
百度云：云端第二备份
```

PC 和移动硬盘直接使用 FreeFileSync 工具自动同步，百度云和 PC 直接通过百度云备份功能自动同步。

百度云我有 4T 的容量，不过考虑到速度和安全性，暂时没有开启同步。
